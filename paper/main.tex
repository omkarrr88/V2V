% ABSTRACT
% ============================================================
\begin{abstract}
Blind spot accidents during lane changes are a major cause of urban traffic collisions. Current driver-assistance systems using radar or cameras fail when other vehicles block the line of sight. This paper presents a Vehicle-to-Vehicle (V2V) Blind Spot Detection system that uses wireless communication between vehicles to detect nearby cars even when they cannot be seen. Our system uses only 5 basic data fields from the standard vehicle safety message (position, speed, acceleration, and braking) — far fewer than the 13 fields required by previous systems. We introduce a Collision Risk Index (CRI) with a severity gate that prevents false alarms by requiring at least one danger signal to be genuinely high before triggering a warning. The system was tested using 146,051 simulated vehicle interactions on a real road network (Atal Bridge, Navi Mumbai) using traffic simulation software. Evaluated against a kinematic near-miss proxy (see Section~\ref{sec:results}), it achieved an AUC of 0.9869. A supporting machine learning model (XGBoost) provides complementary pattern-based detection with an AUC of 0.7725.
\end{abstract}

% I. INTRODUCTION

\section{Introduction}

Road traffic injuries cause about 1.19 million deaths worldwide every year, making it the leading cause of death for people aged 5--29~\cite{who2023}. Many of these accidents happen when drivers change lanes without seeing vehicles in their blind spots. In India, dense and mixed traffic on urban roads makes this problem even worse, often leading to multi-vehicle crashes~\cite{morth2023}.

Blind spots are areas around a vehicle that the driver cannot see through mirrors or windows. The car's pillars and limited mirror angles create these hidden zones. While modern driver-assistance systems use sensors like radar and cameras to detect vehicles in blind spots, these sensors fail when large vehicles physically block the view — which happens frequently in heavy traffic~\cite{biswas2006}.

Vehicle-to-Vehicle (V2V) communication solves this problem by letting vehicles talk directly to each other using radio signals. These radio signals pass through obstacles, so a vehicle can know about nearby cars even when they are hidden behind a truck or bus. Each vehicle continuously broadcasts its location, speed, and other data using a standardized message called the Basic Safety Message (BSM)~\cite{sae_j2735,sae_j2945}. This message is sent 10 times per second and can reach vehicles up to 300 metres away~\cite{biswas2006}. Two main wireless technologies support this: DSRC (Dedicated Short Range Communications) using IEEE 802.11p~\cite{ieee_80211p}, and C-V2X (Cellular Vehicle-to-Everything) using cellular networks~\cite{3gpp2016}.

Previous V2V blind spot detection systems needed up to 13 data fields from the safety message, including heading direction, turn signals, and yaw rate~\cite{mousa2021}. This creates compatibility problems because older vehicles may not support all these fields. Our system uses only 5 fields: GPS position ($x$, $y$), speed ($v$), acceleration ($a$), and braking ($d$). Missing information like heading direction is calculated from the vehicle's position history.

The main contributions of this work are:
\begin{enumerate}
  \item \textbf{Severity-gated Collision Risk Index.} A new risk formula that requires at least one physical danger signal (braking distance or collision timing) to be genuinely high before triggering a warning. This prevents false alarms caused by small, harmless signals adding up.

  \item \textbf{Minimal 5-field BSM.} The system works with only 5 basic data fields, eliminating the need for 13 fields required by previous systems — making it compatible with all types of vehicles.

  \item \textbf{Dual physics-AI architecture.} A machine learning model (XGBoost) works alongside the physics-based model to catch unusual danger patterns, while the transparent physics model remains the primary decision-maker.

  \item \textbf{Realistic testing.} The system was tested on a real road network under challenging conditions including traffic signal violations and narrow hilly roads, with simulated communication failures.
\end{enumerate}

The rest of this paper covers: related work (Section~\ref{sec:related}), system architecture (Section~\ref{sec:architecture}), the mathematical model (Section~\ref{sec:math}), scenario testing (Section~\ref{sec:scenario}), experimental setup (Section~\ref{sec:setup}), results (Section~\ref{sec:results}), deployment (Section~\ref{sec:deployment}), discussion (Section~\ref{sec:discussion}), and conclusion (Section~\ref{sec:conclusion}).

% II. RELATED WORK

\section{Related Work}
\label{sec:related}

\subsection{Radar and Camera-Based Blind Spot Detection}
Current commercial blind spot systems use radar sensors mounted on rear bumpers to detect nearby vehicles. These work well on open roads but have limitations in dense traffic, where large vehicles can block or deflect radar signals. A radar system cannot detect a car approaching from behind a truck. V2V communication overcomes this because radio signals pass through vehicles.

\subsection{V2V Communication Standards and the BSM}
V2V systems use the SAE J2735 standard~\cite{sae_j2735} for message formatting and IEEE 802.11p~\cite{ieee_80211p} for wireless transmission. The Basic Safety Message (BSM) is the core data packet that vehicles broadcast, containing their location, speed, and other safety data. The SAE J2945/1 standard~\cite{sae_j2945} requires this message to be sent 10 times per second. Newer cellular V2X systems~\cite{3gpp2016} support the same message format. Our system uses only 5 fields from the BSM Part~I (position, speed, acceleration, braking), making it work with basic and affordable on-board communication units~\cite{festag2014}.

\subsection{Risk Assessment for V2V Safety}
Previous approaches to collision risk estimation have used various methods. Time-to-Collision (TTC) is a common metric used in Euro NCAP safety testing~\cite{euroncap2019}, but it becomes unreliable when vehicles weave laterally. Early V2V blind spot systems used shared GPS coordinates~\cite{rawashdeh2008}. Kim~\textit{et~al.}~\cite{kim2015} developed a threat assessment model using V2V data, and Mousa~\textit{et~al.}~\cite{mousa2021} created a V2V blind spot system using 13 BSM fields.

A common problem with existing risk models is that multiple small, harmless risk values can add up and trigger false warnings. Our severity gate prevents this by requiring at least one danger signal to be genuinely high before raising an alarm.

\subsection{Machine Learning in Transportation Safety}
Machine learning models like XGBoost~\cite{chen2016xgboost} work well for structured safety data. However, using a purely AI-based system for safety decisions raises concerns because it is hard to understand why the model made a specific decision. Oversampling methods like SMOTE~\cite{smote2002} help handle the fact that dangerous events are rare in safety data. In our system, the AI model serves only as a backup — the transparent physics model makes the primary safety decisions.

\section{System Architecture}
\label{sec:architecture}

The system is organized as a four-layer pipeline (Fig.~\ref{fig:architecture}):

\textbf{Layer~1 --- Simulation \& Environment.}
Eclipse SUMO~\cite{sumo2018} generates realistic vehicle movements at 10 updates per second. Dangerous scenarios (traffic signal violations and narrow road conditions) are injected to test the system under stress.

\textbf{Layer~2 --- Communication \& Perception.}
Each vehicle's data is packaged into a 5-field BSM. GPS noise (1.5~m uncertainty) and random packet drops (using a Gilbert-Elliott channel model) are added to simulate real-world communication conditions. When a message is lost, the system predicts where the vehicle should be using its last known speed and direction.

\textbf{Layer~3 --- Intelligence Kernel.}
Received messages are processed by two parallel paths:
\begin{itemize}
  \item \textbf{Physics model (primary):} Calculates the Collision Risk Index (CRI) using mathematical formulas based on vehicle positions, speeds, and braking distances.
  \item \textbf{AI model (secondary):} An XGBoost classifier trained on 18 features derived from the BSM data, providing backup pattern detection.
\end{itemize}

\textbf{Layer~4 --- Analytics \& Presentation.}
A Streamlit dashboard displays real-time V2V connections, risk scores per vehicle, and comparison charts.

% IV. MATHEMATICAL MODEL

\section{Mathematical Model}
\label{sec:math}

This section describes the formulas used to calculate collision risk. The system takes in 5 data fields from each nearby vehicle and produces a single risk score between 0 (safe) and 1 (critical danger).

\subsection{5-Field BSM Input}
The system uses only 5 values from each vehicle's safety message:
\begin{equation}
  \mathrm{BSM} = \{x,\; y,\; v,\; a,\; d\}
\end{equation}
where $x,y$ are the vehicle's GPS coordinates (converted to metres), $v$ is speed, $a$ is acceleration, and $d$ is braking strength.

The vehicle's heading direction is calculated from its recent positions:
\begin{equation}
  \theta(t) = \atan2\!\bigl(y(t)-y(t-1),\; x(t)-x(t-1)\bigr)
\end{equation}
At very low speeds (below 0.5~m/s), GPS noise makes this calculation unreliable, so the system holds the last known heading:
\begin{equation}
  \theta(t) = \theta(t-1) \quad \text{if } v(t) < 0.5\;\text{m/s}
\end{equation}
The turning rate is then:
\begin{equation}
  \dot{\theta}(t) = \frac{\theta(t) - \theta(t-1)}{\Delta t}
\end{equation}

\subsection{Coordinate Transformation}
To determine which side of the ego vehicle (the driver's car) a target vehicle is on, we transform the target's position into the ego vehicle's reference frame. In this frame, the lateral axis ($x_{\mathrm{rel}}$) points to the right and the longitudinal axis ($y_{\mathrm{rel}}$) points forward:
\begin{align}
  x_{\mathrm{rel}} &= \sin(\theta_e)\cdot\Delta x - \cos(\theta_e)\cdot\Delta y \\
  y_{\mathrm{rel}} &= \cos(\theta_e)\cdot\Delta x + \sin(\theta_e)\cdot\Delta y
\end{align}
where $\Delta x = x_t - x_e$ and $\Delta y = y_t - y_e$. A positive $x_{\mathrm{rel}}$ means the target is on the right side.

\subsection{Dynamic Blind Spot Zone}
The blind spot zone extends further at higher speeds. The zone length is:
\begin{equation}
  L_{\mathrm{bs}}(v_e) = L_{\mathrm{base}} + \lambda_{\mathrm{scale}}
    \cdot \mathrm{clamp}\!\left(\frac{v_e - v_{\min}}{v_{\max} - v_{\min}},\,0,\,1\right)
\end{equation}
On curves, the blind spot zone shifts laterally. A curvature correction adjusts for this when the vehicle is turning:
\begin{equation}
  x_{\mathrm{corrected}} = x_{\mathrm{rel}} - \frac{y_{\mathrm{rel}}^2\cdot\dot{\theta}_e}{2\cdot v_e}
\end{equation}
A target is considered to be in the blind spot if it falls within the lateral range defined by the ego vehicle's width, lane width, and a 0.5~m tolerance margin.

The resulting CRI distribution (Fig.~\ref{fig:cri_dist}) confirms that the severity gate prevents excessive warnings, with 96.1\% of observations remaining in the SAFE region.

\begin{figure}[htbp]
  \centering
  \includegraphics[width=0.9\columnwidth]{fig2_cri_distribution.png}
  \caption{CRI score distribution across 146,051 simulation timesteps. 96.1\% of observations are in the SAFE region (CRI~$<$~0.30), confirming the severity gate prevents excessive warnings.}
  \label{fig:cri_dist}
\end{figure}

\subsection{Blind Spot Zone Probability}
Since GPS is not perfectly accurate (1.5~m uncertainty), the system calculates the \textit{probability} that a target is in the blind spot rather than making a binary yes/no decision. This is done by integrating a Gaussian distribution over the blind spot boundaries:
\begin{align}
  P_{\mathrm{lat}} &= \Phi\!\left(\frac{x_{\max} - x_c}{\sigma}\right) - \Phi\!\left(\frac{x_{\min} - x_c}{\sigma}\right) \\
  P_{\mathrm{lon}} &= \Phi\!\left(\frac{y_{\max} - y_c}{\sigma}\right) - \Phi\!\left(\frac{y_{\min} - y_c}{\sigma}\right)
\end{align}
where $\Phi$ is the standard normal cumulative distribution function, $(x_c, y_c)$ is the target's corrected position, and $\sigma = 1.5$~m is the GPS uncertainty. The combined probability is:
\begin{equation}
  P_{\mathrm{gps}}(V_t \in Z_{\mathrm{bs}}) = |P_{\mathrm{lat}}| \times P_{\mathrm{lon}}
\end{equation}
A forward-lane filter prevents false alarms from vehicles directly ahead in the same lane.

\subsection{Dead Reckoning Under Packet Loss}
When communication packets are lost, the system predicts where the target vehicle should be using its last known speed and direction (Constant-Acceleration, Constant-Yaw-Rate model):
\begin{equation}
  \hat{x} = x + v\cos(\theta)\,\tau + \tfrac{1}{2}\,a_{\mathrm{net}}\cos(\theta)\,\tau^2
\end{equation}
\begin{equation}
  \hat{y} = y + v\sin(\theta)\,\tau + \tfrac{1}{2}\,a_{\mathrm{net}}\sin(\theta)\,\tau^2
\end{equation}
This prediction is limited to 0.5~seconds; if more than four consecutive packets are lost, the target is dropped to avoid unreliable estimates.

\subsection{Packet Loss Safety Multiplier}
A Packet Loss Ratio (PLR) is tracked over a 10-step window. Higher packet loss increases the risk score to account for uncertainty:
\begin{equation}
  \Gamma_{\mathrm{plr}} = 1 + 0.30 \cdot \mathrm{PLR}_{\mathrm{window}}
\end{equation}

\subsection{Deceleration Risk ($R_{\mathrm{decel}}$)}
This measures whether the target vehicle can stop in time. First, the maximum braking capability is calculated:
\begin{equation}
  a_{\max} = \mu\cdot g + \frac{\rho\cdot C_d\cdot A_f\cdot v_t^2}{2\cdot M_t}
\end{equation}
where $\mu$ is road friction, $g$ is gravity, and the second term accounts for air resistance. The total stopping distance includes a 1.2~second reaction time~\cite{aashto2018}:
\begin{equation}
  D_{\mathrm{stop}} = v_t\cdot T_{\mathrm{react}} + \frac{v_t^2}{2\cdot a_{\max}}
\end{equation}
The risk grows exponentially as the stopping distance approaches the actual gap between vehicles:
\begin{equation}
  R_{\mathrm{decel}} = \mathrm{clip}\!\left(
    \exp\!\left(-K_{\mathrm{brake}} \cdot \frac{d_{\mathrm{gap}} - D_{\mathrm{stop}}}{D_{\mathrm{stop}}}\right),\; 0,\; 1
  \right)
\end{equation}
where $K_{\mathrm{brake}} = 1.50$. When the gap exactly equals the stopping distance, risk is 1.0 (maximum); as the gap grows, risk drops toward zero.

The distributions of all three risk components across the simulation are shown in Fig.~\ref{fig:risk_components}.

\begin{figure}[htbp]
  \centering
  \includegraphics[width=0.9\columnwidth]{fig5_risk_components.png}
  \caption{Distributions of the three CRI risk components. High-risk values are rare, consistent with the 2.27\% positive event rate in the simulation.}
  \label{fig:risk_components}
\end{figure}

\subsection{Time-to-Collision Risk ($R_{\mathrm{ttc}}$)}
This measures how soon a collision could happen. The system solves the equation
$a_{\mathrm{rel}}\,\tau^2 + v_{\mathrm{rel}}\,\tau + x_{\mathrm{rel}} = 0$
to find the time until the two vehicles would meet (TTC). The risk is:
\begin{equation}
  R_{\mathrm{ttc}} = \begin{cases}
    1.0
      & \text{if } \mathrm{TTC} \le 4.0\;\text{s} \\[4pt]
    \left(\dfrac{4.0}{\mathrm{TTC}}\right)^{\!2}
      & \text{if } 4.0\;\text{s} < \mathrm{TTC} \le 8.0\;\text{s} \\[6pt]
    0
      & \text{otherwise}
  \end{cases}
\end{equation}
A parallel calculation checks for side collisions using the lateral gap between vehicles. The combined TTC risk takes the higher of the two values.

\subsection{Intent Risk ($R_{\mathrm{intent}}$)}
Without turn signal data, the system tries to estimate whether the ego vehicle is drifting sideways toward another lane:
\begin{equation}
  R_{\mathrm{intent}} = 0.6 \cdot \min\!\left(1,\; \frac{\max(0,\, v_{\mathrm{lat,toward}})}{1.0\;\text{m/s}}\right)
\end{equation}
However, testing showed that predicting lane-change intent without turn signals is unreliable. Therefore, this component is disabled in the final system ($\gamma=0$).

\subsection{Collision Risk Index with Severity Gate}
The final Collision Risk Index (CRI) combines all components into a single score:
\begin{equation}
  \mathrm{CRI} = \mathrm{clip}\!\Big(
    P_{\mathrm{gps}} \cdot \max(R_{\mathrm{decel}},R_{\mathrm{ttc}})
    \cdot \bigl[\alpha R_{\mathrm{decel}} + \beta R_{\mathrm{ttc}}
               + \gamma R_{\mathrm{intent}}\bigr]
    \cdot \Gamma_{\mathrm{plr}},\;0,\;1\Big)
\end{equation}

The key innovation is the $\max(R_{\mathrm{decel}},R_{\mathrm{ttc}})$ \textbf{severity gate}. In traditional additive risk models, multiple small harmless values can add up and trigger a false alarm. The severity gate prevents this by requiring at least one physical danger signal (either braking distance or collision timing) to be genuinely high before the overall score can escalate. The optimized weights are: $\alpha=0.20$ (deceleration), $\beta=0.80$ (time-to-collision), and $\gamma=0.00$ (intent, disabled).

\subsection{Alert Levels}
The system triggers four alert levels based on CRI:
\begin{itemize}
  \item \textbf{SAFE} (CRI $< 0.30$): No danger detected
  \item \textbf{CAUTION} (CRI $\ge 0.30$): Vehicle detected in blind spot
  \item \textbf{WARNING} (CRI $\ge 0.60$): Close proximity or approach
  \item \textbf{CRITICAL} (CRI $\ge 0.80$): Imminent collision risk
\end{itemize}
To prevent alerts from flickering on and off, the system requires 3 consecutive steps above a threshold before upgrading, and a small buffer band (0.05) before downgrading. Left and right sides are monitored independently.

\section{Scenario Modelling}
\label{sec:scenario}

To test the system under realistic danger conditions, two conflict scenarios and a communication failure model were implemented.

\subsection{Traffic Signal Violation (TSV)}
When a vehicle approaches an intersection during a red light, there is a 30\% chance it will ignore the signal and continue at full speed. This creates dangerous crossing conflicts that test the system's ability to detect side-impact collision risks.

\subsection{Hilly Narrow Road (HNR)}
On selected road segments, lane width is reduced to 2.8~metres and road friction is lowered to 0.55 (simulating wet or hilly conditions). Some vehicles (10\% probability) attempt aggressive overtaking. This tests the system's ability to handle reduced braking capability and tight lateral spaces.

\subsection{Communication Packet Loss}
Real-world wireless signals can be interrupted by buildings, other vehicles, or interference. This is modelled using a Gilbert-Elliott two-state channel~\cite{gilbert1960}: in the good state, only 1\% of packets are lost; in the bad state, 50\% are lost. The system spends about 9.1\% of the time in the bad state, creating realistic multi-packet outages that test the dead-reckoning prediction.

The effect of these scenarios on risk levels is shown in Fig.~\ref{fig:scenario_cri}.

\section{Experimental Setup}
\label{sec:setup}

\subsection{Simulation Environment}
All experiments use Eclipse SUMO~\cite{sumo2018} traffic simulator running at 10~Hz (10 updates per second). The road network is a 539-edge map of the Atal Bridge corridor in Navi Mumbai, India, extracted from OpenStreetMap. The simulation runs for 360 seconds with up to 163 simultaneous vehicles, producing 146,051 total observations. All system parameters are listed in Table~\ref{tab:params}.

\subsection{Road Network}
The Atal Bridge network includes multi-lane roads, controlled intersections, and highway ramps — significantly more complex than the simple highway segments commonly used in V2V research. The simulated traffic models Indian conditions: high vehicle density, mixed vehicle types (sedans, SUVs, trucks), and frequent lane changes.

\subsection{AI Model Configuration}
The XGBoost classifier~\cite{chen2016xgboost} serves as a backup detection layer alongside the physics model. From the 5 BSM fields, 18 features are computed including speed, acceleration, braking status, gap distance, relative speed, and scenario indicators. Because dangerous events are rare (only 354 CRITICAL samples out of 146,051), SMOTE oversampling~\cite{smote2002} is used to balance the training data. The model uses 5-fold time-series cross-validation with a 75\%/25\% train/test split.

\subsection{Overall Performance}

\subsection{Ground Truth Definition}
Since the simulation does not produce actual collision labels, a near-miss event is defined as ground truth: a positive event is flagged when a target is simultaneously in the blind spot, within 1.0~m gap or 2.0~s TTC, with relative speed above 0.5~m/s, and ego speed above 2.0~m/s. SUMO-reported physical collisions (27 events) are also treated as authoritative positives. This yields a positive rate of 2.37\% (3,465 events out of 146,051 observations).

\subsection{ROC Analysis}
Fig.~\ref{fig:roc} shows the ROC curves for all four methods. The V3.0 physics model achieves the highest AUC of 0.9869, showing strong ability to distinguish safe conditions from genuine blind-spot hazards. The XGBoost model achieves 0.7725 AUC. The physics model also achieves an AUPRC (Area Under Precision-Recall Curve) of 0.677, far above the random baseline of 0.024.

\subsection{AI Model Results}
The XGBoost model achieves 80.8\% overall accuracy on the holdout test set. Because the dataset is 96\% SAFE-class, a trivial always-SAFE baseline would score higher; the model's value lies in detecting non-SAFE events. Table~\ref{tab:ai_perf} shows per-class performance. The model successfully detects CAUTION (65\% recall) and WARNING (57\% recall) events, but fails on the CRITICAL class (0\% recall) due to extreme rarity — only 13 CRITICAL test samples exist. The low precision for CAUTION (0.12) and WARNING (0.11) reflects inherent difficulty distinguishing intermediate risk levels; however, these errors are conservative (over-alerting rather than missed detections).

% VIII. REAL-WORLD DEPLOYMENT PATHWAY
% ============================================================
\section{Real-World Deployment Pathway}
\label{sec:deployment}

Deploying this system on real vehicles requires only a standard DSRC or C-V2X On-Board Unit (OBU) that can transmit the 5-field BSM. Because the system uses only position, speed, and acceleration values, it does not need access to the vehicle's internal computer network for turn signals or steering data. This makes it compatible with existing aftermarket V2V communication kits.

For embedded hardware deployment, the CRI logic has been packaged as a ROS2 node that converts GPS coordinates into the local coordinate frame used by the physics engine. A single processing cycle with 15 simultaneous targets completes in 4.7~ms, well within the 100~ms time budget required by the SAE J2945/1 standard~\cite{sae_j2945}.

The XGBoost model is exported in ONNX format for efficient embedded inference. The physics model runs as the primary decision-maker, while the AI model provides backup detection on a parallel path — maintaining safety traceability.

\textbf{Current status:} All results in this paper are from simulation. Hardware testing on physical vehicles is planned as future work. The ROS2 wrapper and ONNX export have been validated in software-only testing.

% ============================================================
% IX. DISCUSSION
% ============================================================
\section{Discussion}
\label{sec:discussion}

The main finding is that the transparent physics model (AUC~=~0.9869) significantly outperforms the machine learning model (AUC~=~0.7725) for blind-spot risk assessment, even though both use the same 5-field BSM input. The severity gate — requiring at least one physical danger signal to be genuinely high — is the key to this result. The ablation study confirms: removing the severity gate by adding the unreliable intent component ($\gamma=0.20$) reduces F1 by 44\%.

The XGBoost model provides value as a pattern-based backup for CAUTION (65\% recall) and WARNING (57\% recall) detection, but fails on CRITICAL events due to extreme class rarity (only 54 samples in 146,051 observations). Its feature importance shows it relies on physical measurements rather than scenario labels, suggesting it can generalize with more training data.

The system maintains consistent performance across normal traffic, traffic signal violations, and narrow road scenarios ($\Delta$AUC~$<$~0.002), and across five independent random traffic seeds ($\sigma=0.0007$).

\subsection{Limitations}
\begin{enumerate}
  \item \textbf{Simulation only.} Results are from traffic simulation. Real-world V2V conditions may differ.
  \item \textbf{Proxy ground truth.} The near-miss definition shares inputs with the CRI model, potentially inflating measured accuracy. Testing against real collision data is needed.
  \item \textbf{No pedestrians or cyclists.} The system only detects vehicles with V2V capability.
  \item \textbf{Single road network.} Testing was done on one road network. More networks should be tested.
  \item \textbf{No turn signals.} The 5-field BSM constraint means lane-change intent cannot be reliably predicted.
  \item \textbf{Insufficient CRITICAL samples.} The simulation produced only 54 CRITICAL-class events (CRI~$\geq$~0.80) out of 146,051 observations, preventing the AI model from learning this class. Longer simulations or targeted scenario injection are needed.
  \item \textbf{No security measures.} The system trusts all received data. Fake or manipulated messages could cause incorrect risk assessments.
\end{enumerate}

% ============================================================
% X. CONCLUSION
% ============================================================
\section{Conclusion}
\label{sec:conclusion}

This paper presented a V2V Blind Spot Detection system that uses only 5 basic data fields from the standard vehicle safety message. By not requiring additional data like turn signals or yaw rate, the system works with all types of vehicles equipped with DSRC or C-V2X communication.

The core innovation is the Collision Risk Index (CRI) with a severity gate that prevents false alarms by requiring at least one physical danger signal to be genuinely high before triggering a warning. Tested over 146,051 observations on a real road network with injected danger scenarios, the physics model achieved an AUC of 0.9869 — significantly outperforming both traditional TTC (0.8886) and static box (0.8737) detection methods. The complementary XGBoost model provides backup detection for CAUTION and WARNING events but requires more CRITICAL training samples to achieve reliable high-severity detection.

Future work priorities include: (1)~testing on real vehicles with commercial V2V hardware; (2)~evaluation against real collision data; (3)~longer simulations to generate sufficient CRITICAL training samples for the AI model; and (4)~testing on diverse road networks. The lightweight, transparent system provides a strong foundation for cooperative vehicle safety.
